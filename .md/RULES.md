# LevelUp Academy — Архитектурные правила и стандарты

Этот документ закрепляет ключевые правила разработки для бэкенда и фронтенда.

## 📐 Правила бэкенда (из BACKEND-ARCHITECTURE.md и TASKS.md)

1.  **Multi-tenancy (Мультиарендность):**
    *   Все таблицы, относящиеся к данным учебных центров, должны содержать `branch_id` и `organization_id`.
    *   Каждый запрос должен быть изолирован в пределах `req.scope.organizationId` и `branchId` (если применимо).
    *   Никогда не запрашивайте данные без фильтра по организации.

2.  **Управление архивацией и удалением:**
    *   **Мягкое удаление:** Использовать колонку `deleted_at`. Запросы должны фильтровать `deleted_at IS NULL`.
    *   **Архивация (`is_archived`):** Сущность доступна только для чтения (мутации запрещены и должны возвращать `403 Forbidden`).

3.  **Финансы и платежи:**
    *   Используется сплит-платёж: один счёт (`invoice`) может быть оплачен несколькими транзакциями с общим `split_batch_id`.
    *   Любое изменение баланса коинов осуществляется исключительно через функцию `changeCoins()`. Прямые UPDATE-запросы к `coin_balance` запрещены.

4.  **Фоновые очереди:**
    *   Все внешние интеграции (отправка Telegram-уведомлений, SMTP-письма, SMS) выполняются асинхронно через BullMQ (очередь `notifications`). HTTP-запросы никогда не должны напрямую ожидать выполнения внешних сетевых вызовов.

## 🎨 Правила фронтенда (из FRONTEND-DESIGN-SYSTEM.md)

1.  **Цветовая палитра (тема Light):**
    *   `--bg`: `#F6FBEA` (светлый салатовый фон)
    *   `--surface`: `#FFFFFF` (карточки, белые с тенями)
    *   `--text`: `#1D2417` (основной графитово-зеленый текст)
    *   `--text-muted`: `#5E6E52` (второстепенный текст)
    *   `--accent` / `--primary`: `#C6FF34` (фирменный лайм)
    *   `--accent-ink` / `--primary-content`: `#141B10` (текст НА лайме, темный)
    *   `--sidebar`: `#1D2417` (темный сайдбар)

2.  **Компоненты и стилизация:**
    *   Карточки должны иметь тень: `box-shadow: 0 10px 28px rgba(29,36,23,.14), 0 2px 6px rgba(29,36,23,.08)`.
    *   При наведении на карточки они должны плавно приподниматься: `hover:-translate-y-0.5 transition-all`.
    *   В таблицах использовать салатовую шапку (`#eef7e4`) и подсветку строк при наведении (`#f6faef`).
    *   Все формы должны валидироваться с помощью `react-hook-form` + `zod` перед отправкой.
